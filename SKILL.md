## 一、 项目整体核心技能 (Project Overall Skills)

- **工业级自动化架构设计**：主导设计并实现了基于 Python + Pytest 的五层逻辑架构（Case-Flow-API-Engine-Data），实现了业务逻辑与底层技术的深度解耦。
- **复杂金融业务建模**：针对 100+ 接口的长链路金融业务（开户、放款、还款），构建了闭环自动化测试方案，支持 40-90 个动态参数的复杂入参处理。
- **高性能数据驱动引擎**：设计并开发了基于 YAML 的数据驱动引擎，引入**懒加载（Lazy Loading）**与 **LRU 缓存机制**，解决了大规模测试数据加载时的内存性能瓶颈。
- **自定义业务协议断言**：针对金融系统非 2xx 的自定义状态码（如 9888x 系列）设计了灵活的断言映射机制，能够精准识别"权限拦截"与"业务互斥"逻辑。
- **环境感知与稳定性治理**：构建了数据库健康预检（Health Check）与自动重连机制，通过异步轮询断言（Polling Assertion）解决了金融系统后端 MQ 延迟导致的数据不一致问题。

## 二、 模块级细分技能 (Module-Specific Skills)

按照项目目录结构，各模块对应的核心技术点如下：

### 1. `core/` (核心引擎层)

- **TemplateManager**：利用 Python 高阶函数实现 YAML 模板的单例懒加载，配合磁盘 I/O 优化，提升了 100+ 配置文件场景下的启动速度。
- **DataEngine**：开发了基于正则匹配的动态标签解析器，支持 `${rand_str}`, `${choice}` 等指令，实现了 90 多个接口字段的自动化高保真随机填充。
- **RequestWrapper**：深度封装 `requests` 库，实现了请求/响应的全局拦截器，自动处理金融级加解密、签名校验及自定义状态码的分类放行。
- **GlobalContext**：利用单例模式构建线程安全的全局上下文池，解决了长链路业务中跨接口变量（如申请单号、Token）的实时传递与生命周期管理。

### 2. `api/` & `business/` (原子接口与业务流)

- **API Object Pattern (AOP)**：应用接口对象模式封装原子操作，确保了 100+ 接口契约变更时"一处修改，全局生效"。
- **业务编排逻辑**：通过封装 `flows` 模块，实现了复杂业务场景的逻辑抽象，支持从"授信"到"放款"全路径的自动化闭环执行。

### 3. `data/` (数据与资源层)

- **YAML DSL 设计**：设计了结构化的数据描述语言，通过标签化定义字段属性（类型、长度、枚举），实现了非代码人员对 40+ 字段入参的低成本维护。
- **公共字典治理**：通过 `common_dicts` 统一管理全行级金融枚举值，降低了跨接口测试数据的冗余度。
- **YAML Schema 校验**：开发了基于 JSON Schema 的结构校验器，自动检测字段类型错误、必填项缺失等问题。
- **测试数据版本化**：基于 Git 实现 YAML 模板版本化存储，支持快速回滚与 Diff 对比，确保多环境数据一致性。
- **敏感字段脱敏**：构建了身份证、银行卡、手机号等金融敏感字段的自动脱敏机制，保障测试数据合规性。

### 4. `tests/` (执行与固件层)

- **Pytest 高阶应用**：深度定制 `conftest.py`，利用 `fixture` 的分层作用域（Session/Class/Function）管理数据库连接与测试现场的初始化。
- **失败自愈机制**：设计了失败用例的自动快照捕获与数据清理（Cleanup）策略，确保了 100+ 接口在持续集成过程中的幂等运行。

### 5. `utils/` (工具与工厂层)

- **DB 鲁棒性封装**：封装了支持自动重连的 `DBClient`，并在 SQL 断言中引入了超时判定与软硬断言结合机制。
- **IdentityFactory**：针对金融监管要求，开发了符合算法规则的"三要素"（姓名、证件、手机号）联动生成器，绕过了后端的合规校验拦截。

### 6. `config/` & 报告 (环境与可视化)

- **多环境适配方案**：通过配置中心动态加载不同环境的 `env.yaml`，实现了 Dev/Test 环境的无缝切换。
- **Allure 深度定制**：在报告中集成了业务链路追踪图、JSON Diff 差异高亮及 TraceID 关联，将排障效率提升了 50% 以上。
